Java Socket Programming

This tutorial covers the core socket operations you need for the pub-sub broker: listening for connections, reading HTTP requests, and writing HTTP responses.

Creating a Server Socket

A ServerSocket binds to a port and listens for incoming TCP connections.

ServerSocket serverSocket = new ServerSocket(8080);
System.out.println("Listening on port 8080");

This creates a TCP socket bound to port 8080. The OS will queue incoming connections until you accept them.

The Accept Loop

Call accept() to wait for the next client connection. It blocks until a client connects and returns a Socket representing that connection.

while (true) {
    Socket clientSocket = serverSocket.accept();
    // handle clientSocket ...
}

Reading from a Socket

Wrap the socket's input stream in a BufferedReader to read text line by line. This is ideal for HTTP headers, which are line-delimited.

BufferedReader in = new BufferedReader(
    new InputStreamReader(clientSocket.getInputStream()));

String line;
while ((line = in.readLine()) != null && !line.isEmpty()) {
    System.out.println(line);
    // parse headers here
}
Tip: HTTP headers end with a blank line (\r\n\r\n). The readLine() method strips the line terminator, so an empty header section gives you an empty string "". The condition !line.isEmpty() detects this boundary.

Reading a Fixed Number of Bytes (Request Body)

HTTP request bodies (like a POST with a JSON payload) have a length specified by the Content-Length header. You cannot use readLine() for the body because the body may not end with a newline, and binary data would be corrupted. Instead, read exactly that many characters from the BufferedReader:

// After parsing headers, you know contentLength
int contentLength = ...; // parsed from Content-Length header

char[] body = new char[contentLength];
int totalRead = 0;
while (totalRead < contentLength) {
    int read = in.read(body, totalRead, contentLength - totalRead);
    if (read == -1) break; // connection closed prematurely
    totalRead += read;
}
String bodyStr = new String(body, 0, totalRead);
Tip: A single call to read() may not return all the bytes you asked for. The while loop ensures you keep reading until you have exactly contentLength characters. This is a common pattern in network programming.

Writing HTTP Responses

Use PrintWriter for the status line and headers (text), and the raw OutputStream for the body (which may be binary or text). Always flush after writing.

OutputStream rawOut = clientSocket.getOutputStream();
PrintWriter headerWriter = new PrintWriter(rawOut, false); // no auto-flush

// Write status line and headers
headerWriter.print("HTTP/1.1 200 OK\r\n");
headerWriter.print("Content-Type: application/json\r\n");
headerWriter.print("Content-Length: " + bodyBytes.length + "\r\n");
headerWriter.print("\r\n");
headerWriter.flush();

// Write body bytes
rawOut.write(bodyBytes);
rawOut.flush();
Tip: Use \r\n (CRLF) for HTTP line endings, not \n. Use print() not println() so you control the line endings exactly. And always call flush() — otherwise data may sit in a buffer and the client will hang waiting for it.

Why PrintWriter for Headers, OutputStream for Body?

Headers are always ASCII text, so PrintWriter is convenient. But the body could be arbitrary bytes. If you use PrintWriter for the body, character encoding issues can corrupt binary data. The safe pattern is:

  • PrintWriter wrapping the OutputStream — for headers only
  • Flush the PrintWriter before switching to raw OutputStream
  • OutputStream.write(byte[]) — for the body

To get body bytes from a string: byte[] bodyBytes = bodyString.getBytes(StandardCharsets.UTF_8);

Closing Sockets

Always close the client socket when you are done. Use try-with-resources or a try/finally block:

try (Socket clientSocket = serverSocket.accept()) {
    // handle request and send response
} // socket is automatically closed here

Or equivalently with try/finally:

Socket clientSocket = serverSocket.accept();
try {
    // handle request and send response
} finally {
    clientSocket.close();
}

Closing the socket also closes its input and output streams. Do not close the BufferedReader or PrintWriter independently — closing the socket is sufficient and avoids confusing double-close issues.

Putting It Together

Here is a minimal server that reads an HTTP request and sends a fixed response:

ServerSocket serverSocket = new ServerSocket(8080);
while (true) {
    try (Socket client = serverSocket.accept()) {
        BufferedReader in = new BufferedReader(
            new InputStreamReader(client.getInputStream()));
        OutputStream rawOut = client.getOutputStream();
        PrintWriter headerWriter = new PrintWriter(rawOut, false);

        // Read request line and headers
        String requestLine = in.readLine();
        String line;
        while ((line = in.readLine()) != null && !line.isEmpty()) {
            // parse headers if needed
        }

        // Send response
        byte[] body = "{}".getBytes(StandardCharsets.UTF_8);
        headerWriter.print("HTTP/1.1 200 OK\r\n");
        headerWriter.print("Content-Type: application/json\r\n");
        headerWriter.print("Content-Length: " + body.length + "\r\n");
        headerWriter.print("\r\n");
        headerWriter.flush();
        rawOut.write(body);
        rawOut.flush();
    }
}
