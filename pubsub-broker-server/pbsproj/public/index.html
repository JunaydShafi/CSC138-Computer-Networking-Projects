PBS — Pub-Sub Broker Server

An HTTP/1.1 message broker in Java

About This Project

PBS is an HTTP/1.1 server that implements a publish-subscribe message broker. Clients subscribe to topics, publish messages, and poll for delivery — all through standard HTTP requests. You will build the server; the test client (pbc.java) is provided.

API Endpoints

MethodPathHeaders / BodyDescription
POST /subscribe/<topic> X-Client-ID required Subscribe client to a topic
POST /unsubscribe/<topic> X-Client-ID required Unsubscribe client from a topic
POST /publish/<topic> X-Client-ID required; message in body Publish message to all subscribers of topic
GET /messages X-Client-ID required Retrieve and clear pending messages
GET /status — List topics with subscriber counts

Response Format

All responses are JSON with appropriate HTTP status codes:

HTTP/1.1 200 OK Content-Type: application/json Content-Length: 42 Connection: close {"status":"subscribed","topic":"weather"}

Command-Line Usage

Server (pbs.java)

java pbs.java [options] [port]
-q
Quiet mode — suppress per-request logging
[port]
Port number (default: 8080)

Test Client (pbc.java)

java pbc.java [options]
-p <port>
Server port (default: 8080)
-t <n>
Number of topics (default: 3)
-s <n>
Subscribers per topic (default: 2)
-m <n>
Messages per topic (default: 5)

Quick Start

# Terminal 1 — start server java pbs.java 8080 # Terminal 2 — run tests java pbc.java -p 8080

Manual Testing with curl

# Subscribe curl -X POST -H "X-Client-ID: alice" http://localhost:8080/subscribe/weather # Publish curl -X POST -H "X-Client-ID: bob" -d "sunny today" http://localhost:8080/publish/weather # Poll curl -H "X-Client-ID: alice" http://localhost:8080/messages # Status curl http://localhost:8080/status
